# image-post (Folder2Page / EasyMotion v2)

Facebook multi-page auto-content poster with **AI-generated captions**, full
proxy support, and a cross-platform UI. Runs on Windows for local testing
and on Linux (Ubuntu 22.04+) for VPS production.

> Original tool: Folder2Page / EasyMotion. This fork adds:
> - 🤖 Multi-provider AI caption fallback chain (OpenRouter → Gemini → Groq).
> - 🌐 Configurable language per tool clone (default Mexican Spanish).
> - 🏷️ Tool-name / niche / custom-prompt UI.
> - 🐧 Linux folder browser (`/api/list-dir`) so the existing Windows UI works on a headless VPS.
> - 🚀 `scripts/start.sh`, systemd unit, and pm2 ecosystem file for VPS deploy.

---

## 1. What it does

1. You drop image/video files into a **shared content folder** (e.g. via FileZilla).
2. The engine scans the folder, picks the first file alphabetically, **generates a
   caption with AI** (or falls back to the filename), and posts it to one of your
   selected Facebook Pages.
3. The file is deleted after a successful post; failures are moved to a
   `failed/` subfolder for retry / inspection.
4. The engine repeats for the next selected page, with anti-ban gap minutes
   (±30 % randomization), multi-round support, and crash recovery.

## 2. Tech stack

| Layer | Tech |
|---|---|
| Backend | Node.js 18+, Express, SQLite (`sqlite3` package) |
| Frontend | React 18 + Vite (built into `frontend/dist/`, served statically) |
| AI | OpenRouter, Google Gemini, Groq — all free-tier compatible |
| Proxies | HTTP / HTTPS / SOCKS5 (`https-proxy-agent`, `socks-proxy-agent`) |
| Facebook | Graph API v19, no app credentials needed (user access token only) |

## 3. Quick start

### Local (Windows or Linux)

```bash
git clone https://github.com/ahmedhasson763-ops/image-post.git
cd image-post

# install backend deps
npm install

# install + build frontend
cd frontend && npm install && npm run build && cd ..

# start (port 5016 by default — change in .env)
npm start
```

Open <http://localhost:5016> and:

1. Paste a long-lived Facebook user access token (Dashboard tab).
2. Pick the pages you want to post to.
3. Type or browse a content folder (e.g. `/root/imagestool1content`).
4. Go to **🤖 AI Captions** → add at least one API key → toggle **Enable AI captions ON**.
5. Hit **🚀 Start**.

### VPS (Ubuntu 22.04)

See [`DEPLOY.md`](DEPLOY.md) for full instructions including UFW, systemd, pm2, and FileZilla setup.

## 4. AI caption fallback chain

Captions are generated by the first AI provider whose request succeeds, ordered
by **priority** (lower = tried first). On any failure (rate-limit, network,
empty response), the engine automatically falls through to the next provider.
If all providers fail and `Fallback to filename` is ON, the filename (without
extension) is used as the caption, preserving the original Folder2Page behavior.

Supported providers and recommended free models:

| Provider | Free models | Vision? |
|---|---|---|
| **OpenRouter** | `google/gemini-2.0-flash-exp:free`<br>`meta-llama/llama-3.2-11b-vision-instruct:free`<br>`mistralai/mistral-7b-instruct:free` | ✅ / ✅ / ❌ |
| **Google Gemini** | `gemini-2.0-flash-exp`<br>`gemini-1.5-flash-8b`<br>`gemini-1.5-flash` | ✅ all |
| **Groq** | `llama-3.2-11b-vision-preview`<br>`llama-3.3-70b-versatile`<br>`llama-3.1-8b-instant` | ✅ / ❌ / ❌ |

The default prompt enforces **Facebook Community Standards**: no clickbait, no
ALL-CAPS, no engagement-bait, max 1–2 emojis, max 3 hashtags at the end, one
open-ended question. You can override it per tool with a custom prompt template
that supports `{language}`, `{niche}`, `{filename}`, `{has_image}` placeholders.

## 5. Cloning the tool for additional pages / niches

Each tool clone runs on its **own port** with its **own content folder** and
**own AI settings**:

```bash
# On the VPS — clone tool 2:
cp -r /root/imagestool1 /root/imagestool2
cd /root/imagestool2

# Edit .env to use a different port
sed -i 's/PORT=5016/PORT=5017/' .env

# Wipe the imagestool1 database so tool 2 starts fresh
rm -rf data/

# Start
bash scripts/start.sh
```

Then in the UI go to **🤖 AI Captions** and change `tool_name` to `imagestool2`
— the suggested content folder will automatically become `/root/imagestool2content`.

## 6. Proxy support

Already built-in. Go to **🛡️ Proxy** tab → paste your proxies (one per line, any
of these formats):

```
ip:port
ip:port:user:pass
user:pass@ip:port
http://user:pass@ip:port
https://user:pass@ip:port
socks5://user:pass@ip:port
```

Health-check + geo-lookup runs automatically. Either map each Facebook page to
a specific proxy manually, or use **Auto-distribute** (N pages per proxy).

## 10. Security

- **Never commit API keys to git.** All keys are entered in the UI and stored
  only in the local SQLite database (`data/easymotion.sqlite`).
- The UI **masks API keys** (first 6 + last 4 chars) when listing providers.
  Sending the masked value in a PUT request leaves the key unchanged.
- The Facebook user access token is stored in plain text in SQLite —
  protect the `data/` directory (set `chmod 700 /root/imagestool1/data`).

If you previously shared keys in chat or other public channels, **rotate them**
before deploying.
